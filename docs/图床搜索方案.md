# 向量数据库 + 传统数据库混合架构分析

## 当前架构

```
用户提问
    │
    ├─→ 向量搜索 (FAISS) — 文本 chunks 语义搜索
    ├─→ 关键词搜索 (BM25) — 文本 chunks 精确匹配
    │     └─→ RRF 融合 → 文本结果
    │
    └─→ SQLite LIKE — 图床 caption 模糊匹配
          └─→ 图片结果
```

## 问题

当前图床搜索只用 `caption LIKE '%Figure 16%'`：
- 不支持词根/分词（搜 "PR曲线" 找不到 "precision-recall curve"）
- 不支持排序（LIKE 只能全表扫，无法评分）
- 不支持多字段联合过滤（只搜 caption，不搜上下文）

## 推荐方案：SQLite FTS5 + 关键词扩展

### 架构

```
传统数据库 (SQLite FTS5)               向量数据库 (FAISS)
│                                       │
├─ figures 表（元数据）                  ├─ chunks 表（文本）
│  ├─ id, paper, caption, page_no       │  ├─ text embedding
│  ├─ **extracted_text** ← 新增         │  ├─ metadata (source, section)
│  └─ **tags** ← 新增                   │  └─ BM25 index
│                                       │
├─ figures_fts (全文索引)                │
│  ├─ caption 分词搜索                   │
│  ├─ extracted_text 分词搜索            │
│  └─ BM25 排序                         │
│                                       │
└─ 结果：匹配的图片 + 评分               └─ 结果：文本 chunks + 评分
        │                                       │
        └─────────── RRF 融合 ──────────────────┘
                           │
                        LLM + 图片
```

### 改什么

| 字段 | 来源 | 用途 |
|------|------|------|
| `caption` | PyMuPDF 文本提取 "Figure X. ..." | FTS 精确匹配 |
| `extracted_text` | **新增** — 从图片所在页提取周围段落文字 | FTS 语义扩展（搜 "PR曲线" 也能命中） |
| `tags` | **新增** — 从论文文本提取的关键词标签 | 分类过滤 |
| `page_text` | **新增** — 该页的完整文本内容 | FTS 全文辅助 |

### 怎么实现

```python
# 1. 建表 + FTS 索引
CREATE VIRTUAL TABLE figures_fts USING fts5(
    caption, extracted_text, tags,
    content='figures', content_rowid='id'
);

# 2. 插入数据时自动提取页面文本
def extract_figures(pdf_path):
    doc = fitz.open(pdf_path)
    for page_no in range(len(doc)):
        page = doc[page_no]
        # 原有：截图 + 裁剪
        # 新增：提取该页完整文本
        page_text = page.get_text("text")
        # 用 LLM 或规则从 page_text 提取该 figure 的关键词标签
        tags = extract_tags_from_context(caption, page_text)
        
        # 存入 FTS
        cur.execute(
            "INSERT INTO figures (...) VALUES (...)")

# 3. 搜索
def search_figures(query):
    # FTS5 搜索（支持 BM25 排序、分词、前缀匹配）
    results = cur.execute("""
        SELECT f.*, rank
        FROM figures_fts
        JOIN figures ON figures_fts.rowid = figures.id
        WHERE figures_fts MATCH ?
        ORDER BY rank
        LIMIT 10
    """, (query,))
    return results
```

### SQLite FTS5 的优势

| 能力 | LIKE '%keyword%' | FTS5 |
|------|-----------------|------|
| 分词 | ❌ 只能整词 | ✅ 中文/英文分词 |
| 排序 | ❌ 无 | ✅ BM25 评分 |
| 前缀匹配 | ❌ | ✅ "Fig*" 匹配 Figure/Fig. |
| 多字段 | ❌ 单字段 | ✅ caption + text + tags |
| 速度 | O(n) 全表扫 | O(log n) 倒排索引 |
| 依赖 | 内置 | 内置（Python sqlite3 需开启） |

### 关键决策点（需要你回答）

**1. 文本来源？**
- A: 从同一页 PDF 提取正文文本作为上下文（简单，零额外成本）
- B: 从该论文的向量 chunks 中检索相关文本（更精确，但更复杂）

**2. 关键词标签怎么生成？**
- A: 规则提取（从 caption 里提取数据集名、方法名）
- B: LLM 生成（解析 caption 后让 Kimi 打标签，每张图多花 0.5 秒）
- C: 不做标签，纯靠 FTS 解决

**3. 融合策略？**
- A: 文本和图片分开展示（当前方式，但图片搜索更准）
- B: RRF 融合（文本结果和图片结果按分数混合排序）
- C: 图片结果直接追加到文本结果后面

**4. SQLite FTS5 兼容性？**
Python 自带的 `sqlite3` 模块不一定开启了 FTS5。需要验证：
```python
import sqlite3
c = sqlite3.connect(':memory:')
c.execute("CREATE VIRTUAL TABLE test USING fts5(content TEXT)")
```
如果报错需要换方案（如 PostgreSQL 或 whoosh 纯 Python 全文索引）。
